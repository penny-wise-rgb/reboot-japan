# 提言008：公費執行の「完全デジタル透明化」と多重下請け・利権構造の根絶

---

## 0. 問題意識

定額支給、領収書不要、事後報告——これらはいずれも昭和型のアナログ運用であり、不正の温床である。カラ出張、不透明な経費、談合、天下り、多重下請けによる中抜き。税金を原資とした利権構造は、「制度の穴」ではなく「制度の設計ミス」によって生まれている。

本提言は、税金が1円でも動く全てのプロセスをテクノロジーで自動統制・完全可視化する。国、地方自治体、外郭団体、そして公的資金（補助金等）を受け取る民間組織にいたるまで、例外を認めない。

**「税金を使う者に、1円のブラックボックスも、不当な中抜きも許さない」**

---

## 1. 5つの基幹政策

---

### 政策① 「公費執行DX」の義務化とアナログ精算の法的禁止

全ての政府機関・自治体・外郭団体に対し、国が指定する**「共通公費管理クラウド」**と**「公務用コーポレートカード」**の導入・利用を義務付ける。

**廃止するもの**
- 現金取り扱い
- 紙の領収書を回議書に貼るアナログ事務
- 定額支給・事後報告による精算

**実現すること**
- 決済データが改ざん不可能な形でシステムに直接連携
- カラ出張や不正利用の余地をゼロにする
- 約330万人の公務員のアナログ精算・監査事務を9割削減し、本来の住民サービスへリソースをシフト

#### 「共通公費管理クラウド」の調達自体への自己適用

本提言が最も注意すべき逆説は、**「透明化のためのシステム調達」が最大の不透明案件になるリスク**である。年間25〜30兆円規模の公共調達市場を監視するシステムの開発は、それ自体が巨大な利権案件になりうる。

この逆説を封じるため、以下の4層の対策を同時に実施する。

| レイヤー | 対策 |
|---|---|
| **調達段階** | 政策⑤のルールを自己適用。下請け業者の入札時確定・直接見積もり・E2E決済を、このシステム開発案件に最初から適用する |
| **技術設計** | システムの仕様・コードを完全オープンソース化し、特定ベンダーへの依存を構造的に排除する |
| **導入方法** | 一から作らない。マネーフォワード・Suica法人カード等の既存民間サービスを組み合わせ、新規開発リスクを最小化する |
| **運用監視** | システムの調達・運用・改修を監視する独立した第三者機関（仮称：公費デジタル監視委員会）を別途設ける |

---

### 政策② システムによる「自動ガバナンス（統制）」の導入

出張予約や物品購入の際、役職や職務に応じた制限をシステム側に**事前実装**する。

**制限の例**
- 一般職の新幹線グリーン車・高級ホテルの自動ロック
- 特定業者からのバイネーム購入のアラート
- 予算上限を超えた発注の自動ブロック

人手によるチェックや「事後の言い訳」に頼る性善説ガバナンスを廃止する。不適切な公費支出そのものが**「物理的に実行できない」**仕組みを作る。

---

### 政策③ 公費執行データの「リアルタイム・オープンデータ化」

共通システムを通じて執行された全公費データを、原則として**全てリアルタイムでWeb公開（オープンデータ化）**する。

**例外（別途厳格に法律で列挙）**
- 捜査費
- 外交機密の一部

**実現すること**
- 住民・メディア・AIによる相互監視を可能にする
- 「税金の使い道の透明化」を極限まで高める
- 公費の異常値をAIが自動検知し、アラートを発する仕組みへ

#### 内部統制システムと公開システムの分離設計

執行管理システム（内部統制）と情報公開システム（外部公開）は、セキュリティ要件が根本的に異なる。両者を同一システムで一括処理することは重大なリスクを生む。

**設計の原則：内部と外部を明確に分離する**

```
【内部統制システム】           【公開システム】
執行データをリアルタイム記録  ←  マスキング・匿名化処理  →  国民向けWeb公開
（高セキュリティ・完全記録）                               （個人情報除去済み）
```

公開側に流れるのは、マスキング・匿名化処理を経たデータのみとする。内部の生データへのアクセスは、厳格な権限管理のもとで監査目的に限定する。

---

### 政策④ 補助金・委託費の「エンドツーエンド（E2E）」追跡

国や自治体から民間企業・NPO・外郭団体へ流れる補助金・委託費についても、その使途を同じ共通システム上で管理・執行することを**支給の絶対条件**とする。

```
補助金・委託費の流れ：
国・自治体 → 民間企業・NPO・外郭団体
              ↓
    全ての使途を共通システム上で管理・執行
              ↓
    目的外支出・中間搾取を完全追跡可能にする
```

中間搾取（いわゆる中抜き）や目的外の不透明な支出を完全にトラッキングし、税金を原資とした利権構造を根絶する。

---

### 政策⑤ 入札プロセスの透明化と「多重下請け構造」の法的解体

公共事業・政府案件の入札制度を根本から改革し、以下のルールを**法的に義務付ける**。

#### ルール1：入札時点での下請け業者の確定・開示

応札企業は、入札時点で実際に業務を担当する下請け業者をすべて開示・確定させなければならない。事後の不透明な丸投げや孫請けへの再委託は一切禁止とする。

#### ルール2：下請け自身による直接見積もり

入札金額の内訳のうち、下請け業者が担当する部分については、**下請け業者自身が作成した見積書**をそのまま添付させる。元請け企業による一方的な買い叩きや不当なマージンの上乗せをシステム的に排除する。

#### ルール3：支払金額の透明化とE2E決済

確定した下請け業者へ支払われる金額はあらかじめ契約時に明示され、公費管理クラウドを通じて**中抜きなしで直接支払われる仕組み（E2E決済）**を構築する。

#### ルール4：架空下請け・名義貸しの禁止と入札資格即時停止

実務を行わない架空下請け・名義貸しを**法的に明文禁止**する。入札時点での業者登録後に実際の業務を別業者が担っていることが発覚した場合、元請け企業および当該業者ともに**入札資格を即時停止**する。「名前だけ登録して実務は別業者がやる」という抜け穴を制度上封じる。

| 現行の構造 | 改革後の構造 |
|---|---|
| 元請け → 中間業者 → 下請け（金額不透明） | 発注者 → 下請け（直接・金額固定） |
| 事後の丸投げ・孫請けが横行 | 入札時点で全業者・全金額を確定 |
| 元請けが下請けを自由に買い叩く | 下請け自身の見積書が契約に直結 |
| 架空下請け・名義貸しが横行 | 発覚次第、入札資格を即時停止 |
| 談合・天下りの温床 | 技術力のある末端業者に適正対価 |

## 2. 財政・生産性へのインパクト（推計）

本提言の完全実行により、国家全体で一定規模の公費削減が見込める。ただし以下の数値はいずれも**推計であり、前提条件によって大きく変動する**。特に多重下請け解体による削減効果は、「不当マージン率」の実態を裏付ける公的統計が存在しないため、保守的・楽観的の2シナリオで示す。

### 多重下請け解体による中抜き排除（最大の項目）

年間約25〜30兆円の公共調達市場を前提とした試算。

| シナリオ | 不当マージン率の前提 | 削減効果 |
|---|---|---|
| **保守的試算** | 3〜5%（会計検査院指摘事例等から保守的に推計） | 約0.75兆〜1.5兆円 |
| **中央値試算** | 5〜7% | 約1.25兆〜2.1兆円 |
| **楽観的試算** | 8〜10%（重層下請け実態調査等の上限値） | 約2.0兆〜3.0兆円 |

「不当マージン率」の正確な全体像を示す公的データは現時点では存在しない。本試算は会計検査院の指摘事例・建設業の重層下請き実態調査等を参考にした推計であり、制度導入後の実績データによる精緻化が必要である。

### その他の削減効果

| 項目 | 削減効果 | 備考 |
|---|---|---|
| 公費執行DXによる無駄の防止 | 約2,000億〜3,000億円 | 不適切な旅費・経費の自動ロック |
| 事務コスト（人件費）の削減 | 約1,000億〜1,500億円相当 | 公務員のアナログ精算事務を9割削減 |

### 合計（シナリオ別）

| シナリオ | 合計削減効果 |
|---|---|
| 保守的 | 約1.05兆〜2.1兆円 |
| 楽観的 | 約2.3兆〜3.45兆円 |

## 3. 適用範囲

**「税金が1円でも動く組織・プロセス」はすべて対象**とする。例外の拡大解釈を許さないよう、例外規定の定義自体も法律で厳格に列挙する。

| 対象 | 適用 |
|---|---|
| 国の省庁・機関 | ✅ 全面適用 |
| 都道府県・市区町村 | ✅ 全面適用 |
| 外郭団体・第三セクター | ✅ 全面適用 |
| 補助金・委託費を受ける民間企業 | ✅ 公費受領分に適用 |
| NPO・公益法人（公費受領） | ✅ 公費受領分に適用 |

---

## 4. 前提条件・今後の検討課題

- 共通公費管理クラウドの仕様策定・セキュリティ設計（オープンソース化の範囲と脆弱性管理）
- 既存民間サービス（マネーフォワード等）との連携仕様と調達プロセスの設計
- 公費デジタル監視委員会の組織設計・独立性の担保・権限範囲
- 内部統制システムと公開システムの分離設計の詳細（マスキング基準・匿名化手法）
- オープンデータ化の例外範囲の法的定義（捜査費・外交機密の具体的な線引き）
- 多重下請け構造の解体に伴う既存の元請け企業・中間業者の激変緩和措置
- 入札時点での下請け確定ルールと、緊急工事・災害対応など柔軟性が必要な案件との整合性
- E2E決済における下請け業者の資金繰り（支払いサイト・前払い制度の設計）
- AIによる異常値検知システムの誤検知リスクと異議申し立て手続きの設計
- 地方議会・監査委員会との役割分担（既存の行政監視機能との整合）
- 「不当マージン率」の実態把握に向けた統計整備（制度導入後の実績データ蓄積）
